\subsubsection{RegisterForm Component}

\textbf{Purpose:} Handles new user registration by collecting credentials (first name, last name, email, and password) and submitting them to the API. Upon successful resolution, it populates global state, local storage, and client-side cookies before passing the user to the verification pipeline.

\vspace{0.2cm}
\noindent\textbf{Location:} \texttt{src/pages/Auth/components/RegisterForm.jsx}

\vspace{0.2cm}
\noindent\textbf{Notes:}
\begin{itemize}
    \item Delegates network operations to \texttt{AuthService.register()} to cleanly dispatch form payloads to the backend application.
    \item On a successful API response, injects the generated \texttt{accessToken}, \texttt{refreshToken}, and \texttt{userDetails} directly into the global \texttt{UserContext} via \texttt{setAuth}.
    \item Hardens client-side security by simultaneously saving the \texttt{accessToken}, \texttt{refreshToken}, and \texttt{userDetails} to cookies utilizing \texttt{js-cookie} with strict \texttt{secure: true} and \texttt{sameSite: "Strict"} parameters.
    \item Temporarily commits the user's submitted email address to \texttt{localStorage} under the key \texttt{registeredEmail} to cleanly pass contextual data to the subsequent verification view.
    \item Automatically transfers focus to the \texttt{/verify-email} route, carrying along an explicit success message inside the React Router \texttt{navigate} state configuration.
    \item Optimizes form responsiveness by rendering an inline loading spinner inside the core submission button element and disabling further user interactions while requests are in flight.
    \item Wraps input operations using a reusable \texttt{PasswordToggle} visibility sub-component, accompanied by clear inline validation rules (demanding at least 8 characters, alongside uppercase, lowercase, numeric, and symbolic characters).
    \item Mounts the \texttt{SocialLogin} component at the base of the form view, presenting automated OAuth integrations with third-party networks (Google, Discord, and GitHub).
    \item Gracefully catches system rejections and errors via \texttt{react-toastify}, converting verbose server exceptions into readable text utilizing an isolated \texttt{apiErrorMessage} utility function.
\end{itemize}

\textbf{Purpose:} Processes existing user entry credentials by validating email and password pairs against backend microservices, establishing authenticated states, and presenting a nested operational route to handle password resets.

\vspace{0.2cm}
\noindent\textbf{Location:} \texttt{src/pages/Auth/components/LoginForm.jsx}

\vspace{0.2cm}
\noindent\textbf{Component Props:}

\vspace{0.2cm}
\noindent\textbf{Notes:}
\begin{itemize}
    \item Integrates closely with \texttt{AuthService.login()} to coordinate client authorization requests.
    \item Following token validation, maps server variables into the centralized application \texttt{UserContext} and commits authentication state into secure, protected browser cookies flags.
    \item Accelerates client-side transitions by smoothly redirecting validated user profiles straight onto the \texttt{/overview} platform dashboard page alongside validation notifications.
    \item Provides an embedded recovery pathway: selecting the "Forgot Password?" anchor routes code execution to \texttt{AuthService.forgotPassword()} to trigger a server-side reset email, shifting the routing context to \texttt{/forgot-password}.
    \item Toggles a processing spinner inside the main form button element during active asynchronous processes, disabling subsequent actions to eliminate race conditions or repetitive operations.
    \item Uses an absolute integration layer with \texttt{react-toastify} to catch, map, and output validation flags or server authentication rejections.
    \item Extends application availability metrics by exposing the \texttt{SocialLogin} block, resolving instant federation profiles across Google, Discord, and GitHub identity systems.
\end{itemize}

\begin{lstlisting}
const handleSubmit = async (event) => {
    event.preventDefault();
    setIsLoading(true);

    const formData = new FormData(event.target);
    const loginData = {
      email: formData.get("email"),
      password: formData.get("password"),
    };

    try {
      const responseData = await AuthService.login(loginData);
      const user = responseData.data.user;

      userContext.setAuth({
        accessToken: responseData.data.accessToken,
        refreshToken: responseData.data.refreshToken,
        userDetails: user,
      });

      Cookies.set("accessToken", responseData.data.accessToken, { secure: true, sameSite: "Strict" });
      Cookies.set("refreshToken", responseData.data.refreshToken, { secure: true, sameSite: "Strict" });
      Cookies.set("userDetails", JSON.stringify(user), { secure: true, sameSite: "Strict" });

      const destination = user.role === "ADMIN" ? "/admin/users" : "/overview";
      navigate(destination, { state: { toastMessage: "You logged in successfully." } });
    } catch (error) {
      console.error("Login error:", error);
      toast.error(apiErrorMessage(error, "Failed to login user"));
    } finally {
      setIsLoading(false);
    }
  };
\end{lstlisting}

\subsubsection{useSocialLogin Hook}

\textbf{Purpose:} Extracts the \texttt{ticket} query parameter from the OAuth callback URL, exchanges it for auth tokens via the backend, stores them, and redirects the user accordingly.

\vspace{0.2cm}
\noindent\textbf{Location:} \texttt{src/hooks/useSocialLogin.js}

\vspace{0.2cm}
\noindent\textbf{Behavior:}
\begin{itemize}
    \item Runs once on mount using a \texttt{useRef} guard to prevent duplicate executions in React Strict Mode.
    \item If no \texttt{ticket} is found in the URL, redirects to \texttt{/login} with an error toast.
    \item On success, stores \texttt{accessToken}, \texttt{refreshToken}, and \texttt{userDetails} in both \texttt{UserContext} and browser cookies, then redirects to \texttt{/overview} with a success toast.
    \item On failure, redirects to \texttt{/login} with a generic error toast.
\end{itemize}

\begin{lstlisting}
useEffect(() => {
        if (calledRef.current) return; // run once
        calledRef.current = true;

        const params = new URLSearchParams(location.search);
        const ticket = params.get("ticket");

        if (!ticket) {
        navigate("/login", { state: { toastMessage: "Invalid login callback!" } });
        return;
        }

        const performLogin = async () => {
            try {
                const data = await AuthService.exchangeTicket(ticket);

                const { user, accessToken, refreshToken } = data.data;

                // Set auth in context
                setAuth({ accessToken, refreshToken, userDetails: user });

                // Store tokens and user details in cookies/localStorage
                Cookies.set("accessToken", accessToken);
                Cookies.set("refreshToken", refreshToken);
                Cookies.set("userDetails", JSON.stringify(user));

                navigate("/overview", { state: { toastMessage: "Successfully logged in!" } });
            } catch (err) {
                console.error(err);
                navigate("/login", { state: { toastMessage: "Something went wrong. Try again." } });
            }
        };

        performLogin();

    }, [location.search, navigate, setAuth]);
\end{lstlisting}
